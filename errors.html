Błędy Git i Rozwiązania

Najczęstsze problemy z Git i ich rozwiązania. Znajdź szybko rozwiązanie swojego problemu.

error: src refspec main does not match any

Błąd występuje gdy próbujesz wysłać branch, który nie istnieje lokalnie.

WYSOKI

Rozwiązanie:

1. Sprawdź dostępne branche:
$ git branch
* main
master
2. Upewnij się, że masz commit:
$ git log --oneline
# Jeśli brak commitów, najpierw je utwórz
$ git add .
$ git commit -m "Initial commit"
3. Poprawna komenda push:
$ git push origin main
# Lub jeśli używasz master:
$ git push origin master
Prewencja:
  • • Zawsze sprawdzaj nazwę brancha przed pushem
  • • Użyj git status aby potwierdzić aktualny branch
  • • Upewnij się, że masz co najmniej jeden commit

failed to push some refs to

Zdalne repozytorium ma zmiany, których nie masz lokalnie.

WYSOKI

Rozwiązanie:

1. Najpierw pobierz zmiany:
$ git pull origin main
# Lub użyj rebase dla czystszej historii:
$ git pull --rebase origin main
2. Rozwiąż ewentualne konflikty:
# Git pokaże pliki z konfliktami
$ git status
# Edytuj pliki i rozwiąż konflikty
$ git add .
$ git commit -m "Resolve conflicts"
3. Wyślij swoje zmiany:
$ git push origin main
Alternatywne rozwiązania:
# Jeśli chcesz nadpisać zdalne zmiany (OSTROŻNIE!)
$ git push --force origin main
# Bezpieczniejsza wersja:
$ git push --force-with-lease origin main

Permission denied (publickey)

Problem z autentykacją SSH podczas próby połączenia z zdalnym repozytorium.

ŚREDNI

Rozwiązanie:

1. Sprawdź istniejące klucze SSH:
$ ls -la ~/.ssh
# Powinieneś zobaczyć id_rsa, id_rsa.pub lub similar
2. Jeśli brak kluczy, wygeneruj nowe:
$ ssh-keygen -t ed25519 -C "your-email@example.com"
# Lub dla starszych systemów:
$ ssh-keygen -t rsa -b 4096 -C "your-email@example.com"
3. Dodaj klucz do ssh-agent:
$ eval "$(ssh-agent -s)"
$ ssh-add ~/.ssh/id_ed25519
4. Skopiuj klucz publiczny do GitHub:
$ cat ~/.ssh/id_ed25519.pub
# Skopiuj wynik i dodaj do GitHub → Settings → SSH keys
5. Przetestuj połączenie:
$ ssh -T git@github.com
# Powinieneś zobaczyć wiadomość powitalną

CONFLICT (content): Merge conflict in file.txt

Wystąpił konflikt podczas próby scalenia zmian z różnych źródeł.

WYSOKI

Rozwiązanie krok po kroku:

1. Sprawdź status repozytorium:
$ git status
# Git pokaże pliki z konfliktami
2. Otwórz plik z konfliktem:
# Plik będzie zawierał markery konfliktu:
<<<<<<< HEAD
Twoja wersja kodu
=======
Wersja z brancha feature
>>>>>>> feature
3. Edytuj plik i rozwiąż konflikt:
# Usuń markery konfliktu i zostaw poprawną wersję
# Poprawna wersja kodu po rozwiązaniu konfliktu
4. Dodaj rozwiązany plik:
$ git add file.txt
5. Zatwierdź rozwiązanie konfliktu:
$ git commit -m "Resolve merge conflict in file.txt"
Przydatne narzędzia:
# Użyj mergetool dla graficznego rozwiązywania
$ git mergetool
$ git config --global merge.tool vimdiff

Please tell me who you are

Brak skonfigurowanego user.name i user.email dla commitów.

ŚREDNI

Skonfiguruj dane użytkownika

git config --global user.name "Imię Nazwisko"
git config --global user.email "email@example.com"
# Tylko dla repo (bez --global)
git config user.name "Imię Nazwisko"
git config user.email "email@example.com"

Niewłaściwe końce linii (CRLF/LF)

Różnice CRLF/LF powodują zbędne zmiany plików między systemami.

NISKI

Ustawienia zależnie od OS

# Windows
git config --global core.autocrlf true
# macOS/Linux
git config --global core.autocrlf input
# Wymuś LF per repo
echo "* text=auto eol=lf" > .gitattributes

.gitignore nie działa

Pliki już śledzone nie zostaną zignorowane automatycznie.

NISKI

Przestań śledzić i dodaj do .gitignore

echo "node_modules/" >> .gitignore
git rm -r --cached node_modules
git add . && git commit -m "Respect .gitignore"

remote origin already exists

Dodano już remote o tej nazwie.

ŚREDNI

Zmień lub usuń remote

git remote -v
# Aktualizuj URL
git remote set-url origin https://github.com/user/repo.git
# Albo usuń i dodaj ponownie
git remote remove origin
git remote add origin https://github.com/user/repo.git

Authentication failed for 'https://...'

Błędne hasło lub wymagany Personal Access Token (PAT).

ŚREDNI

Użyj PAT zamiast hasła

# Skonfiguruj helper zapamiętujący poświadczenia
git config --global credential.helper store
# Następny push poprosi o login i PAT
git push

Utwórz PAT z odpowiednimi uprawnieniami (repo, workflow) w ustawieniach platformy (GitHub/GitLab).

Support for password authentication was removed

Serwisy jak GitHub wymagają tokenów lub SSH.

ŚREDNI

Przejdź na SSH lub PAT

# SSH (zalecane)
ssh-keygen -t ed25519 -C "email@example.com"
ssh -T git@github.com
# Lub ustaw remote na HTTPS + PAT
git remote set-url origin https://github.com/user/repo.git

insufficient scopes dla tokenu

Token nie ma wymaganych uprawnień (np. repo).

NISKI

Zaktualizuj zakresy tokenu

Wygeneruj nowy token z wymaganymi scope (np. repo, workflow).

SSL certificate problem

Problemy z weryfikacją certyfikatu HTTPS.

NISKI

Zaufaj właściwemu CA (zalecane)

# Zaktualizuj certyfikaty systemowe/CA
# (Unikaj wyłączania weryfikacji)

W ostateczności (na własne ryzyko): git config http.sslVerify false.

Cannot delete branch 'X' checked out at 'Y'

Nie można usunąć bieżącej gałęzi.

NISKI

Przełącz gałąź, potem usuń

git switch main
git branch -d feature/x
# Wymuś (jeśli trzeba)
git branch -D feature/x

error: failed to push some refs (non-fast-forward)

Zdalny branch zawiera nowe commity.

ŚREDNI

Zsynchronizuj i wypchnij

git pull --rebase origin main
# Rozwiąż konflikty, potem
git push origin main
# Ostatecznie (ostrożnie)
git push --force-with-lease origin main

You are in 'detached HEAD' state

Pracujesz na commicie bez gałęzi.

ŚREDNI

Utwórz nową gałąź z tego miejsca

git switch -c fix/detached
git push -u origin fix/detached

Usunięcie zdalnej gałęzi

Różne składnie dla usuwania remote branch.

NISKI

Poprawne komendy

git push origin --delete feature/x
# lub
git push origin :feature/x

Rebase w toku

Operacja rebase wymaga kontynuacji lub przerwania.

NISKI

Dokończ lub przerwij

git status
# napraw konflikty, następnie
git add .
git rebase --continue
# przerwij
git rebase --abort

Szybkie Rozwiązania

Najczęściej używane komendy do rozwiązywania problemów

Cofnij ostatni commit

# Zachowaj zmiany w staging area
git reset --soft HEAD~1
# Przenieś zmiany do working directory
git reset --mixed HEAD~1
# Usuń zmiany całkowicie
git reset --hard HEAD~1

Użyj --soft aby zachować zmiany w staging area, --mixed aby przenieść do working directory, lub --hard aby usunąć całkowicie.

Wyczyść nieśledzone pliki

# Sprawdź co zostanie usunięte
git clean -n
# Usuń nieśledzone pliki
git clean -f
# Usuń także katalogi
git clean -fd

Użyj git clean aby usunąć nieśledzone pliki. Zawsze sprawdź z flagą -n przed usunięciem.

Przywróć usunięty branch

# Znajdź hash usuniętego commita
git reflog
# Przywróć branch
git checkout -b branch-name HASH
# Lub użyj fsck
git fsck --lost-found

Użyj git reflog aby znaleźć hash usuniętego commita, następnie przywróć branch używając tego hasha.

Zmień ostatni commit

# Zmień wiadomość commita
git commit --amend -m "New message"
# Dodaj więcej plików do ostatniego commita
git add forgotten-file.txt
git commit --amend --no-edit

Użyj git commit --amend aby zmienić ostatni commit. Uważaj przy pracy zespołowej!

Stash - tymczasowe zapisy

# Zapisz zmiany tymczasowo
git stash
# Przywróć ostatnie zmiany
git stash pop
# Lista zapisanych zmian
git stash list

Użyj git stash aby tymczasowo zapisać zmiany i przełączyć się na inny branch.

Cherry-pick - wybierz commit

# Znajdź hash commita
git log --oneline
# Przenieś konkretny commit
git cherry-pick abc123
# Przenieś kilka commitów
git cherry-pick abc123..def456

Użyj git cherry-pick aby przenieść konkretne commity z innego brancha do aktualnego.
